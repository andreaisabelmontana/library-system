About this project

A multi-role library management system. This single HTML file is a self-contained demonstration of the domain model so it can run on GitHub Pages with no build step. The authoritative implementation is the Python backend in this repository.

The real backend (Python)

  • library/ — SQLite-backed package: books, per-book physical copies, members (membership types with loan limits), librarians, loans, fines, and a FIFO reservation queue.
  • Role gating — librarian vs member; unauthorized actions raise PermissionError. Members can search, reserve, and view their own history; librarians check out / return / manage the catalog / waive fines.
  • Loan & fine rules — 14-day loans; overdue fine = rate × days late ($0.25/day by default); borrowing blocked over a $5.00 unpaid-fine cap and over the member's loan limit.
  • Reservations — per-title FIFO hold queue; the head is served when a copy is returned.
  • CSV — catalog and member import/export (RFC-4180).
  • tests/ — pytest suite (16 tests); demo.py runs the full lifecycle.

Run the backend

pip install -r requirements.txt python -m pytest -q python demo.py
